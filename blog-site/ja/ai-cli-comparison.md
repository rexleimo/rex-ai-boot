# もう手動で小红書をやらなくていい！AIで毎日2時間増やす自動化運用術

毎日2時間かけて手動で投稿、いいね、コメント、フォロー...この作業やったことある人ならわかりますよね。

僕は夢獸、一个野生程序员です。之前一个人运营小红书账号，每天除了想内容，还要手动刷榜、互动、回复评论。干了一个月，发现一个问题：**我大部分时间都花在重复劳动上，而不是内容创作上。**

RexCLIで全部自動化しました。

## 自動化内容

効果：今毎日**2時間節約**、この時間を女友達と過ごした方が美味しくない？

### 1. 自動投稿

昔：小红書を開く → 投稿をクリック → 画像を選択 → 文案を書く → 确定をクリック → アップロードを待つ → 確認
今：フォルダに素材を投げる → RexCLIが全部自動化

### 2. 自動いいね・コメント

昔：30分关注リストをスクロール → 手動でいいね → 手動でコメント → 手が疲れる
今：キーワードを設定 → RexCLIが自動巡回、自动いいね、自动コメント

### 3. 自動フォロー返し

昔：新ファンがいる → 手動でフォロー → 手動でプロフィール確認 → 手動で判断
今：新規フォローを自動承認 → プロフィールを自動訪問 → ルールに基づいて自動判断

### 4. データ監視

昔：毎日手動で小红書を開く → データを確認 → テーブルに記録 → トレンドを分析
今：定时自动抓取笔记曝光、点赞、收藏データ → 自動汇总到本地

## コア技術：RexCLIの実現方法

質問：これってただのスクリプトじゃないの？RexCLI有何関係？

良い質問。違いを教える。

### ブラウザ自動化 + コンテキスト記憶

RexCLIは二つのコア問題を解決：

**問題一：ブラウザ操作**
小红書はフロントエンドレンダリングの動的ページ、普通スクリプトでは無理。RexCLIはPlaywright MCPを内蔵、直接ブラウザを操作。

**問題二：コンテキスト丢失**
昔、スクリプトが途中で失敗したら？再起動後从哪里开始？RexCLIはContext DB会话記憶機能があり、脚本中断後可以从断点继续。

### 生活喩え

普通スクリプトは使い捨てティッシュ、RexCLIはスマート家政婦。
