## 自動化内容

効果：今毎日**2時間節約**、この時間を女友達と過ごした方が美味しくない？

### 1. 自動投稿

昔：小红書を開く → 投稿をクリック → 画像を選択 → 文案を書く → 确定をクリック → アップロードを待つ → 確認

### 2. 自動いいね・コメント

昔：30分关注リストをスクロール → 手動でいいね → 手動でコメント → 手が疲れる

### 3. 自動フォロー返し

昔：新ファンがいる → 手動でフォロー → 手動でプロフィール確認 → 手動で判断
今：新規フォローを自動承認 → プロフィールを自動訪問 → ルールに基づいて自動判断

昔：毎日手動で小红書を開く → データを確認 → テーブルに記録 → トレンドを分析
今：定时自动抓取笔记曝光、点赞、收藏データ → 自動汇总到本地

## コア技術：RexCLIの実現方法

質問：これってただのスクリプトじゃないの？RexCLI有何関係？

良い質問。違いを教える。

### ブラウザ自動化 + コンテキスト記憶

RexCLIは二つのコア問題を解決：

**問題一：ブラウザ操作**
小红書はフロントエンドレンダリングの動的ページ、普通スクリプトでは無理exCLIは。RPlaywright MCPを内蔵、直接 реальный ブラウザを操作。

**問題二：コンテキスト丢失**
昔、スクリプトが途中で失敗したら？再起動後从哪里开始？RexCLIはContext DB会话記憶機能があり、脚本中断後可以从断点继续、不会重复操作。

### 生活喩え

普通スクリプトは：**使い捨てティッシュ**、使ったら捨てる。RexCLIは：**スマート家政婦**、働く＋前回の進捗を覚えている。

## よくある落とし穴

### 落とし穴1：アカウント風控

小红書は自動化操作に風控システムがある、1秒に10回操作したらアウト。

**对策**：ランダム遅延を設定、5-30秒ランダムで人間の行動をシミュレート。

### 落とし穴2：ログイン状態消失

長く実行するとCookie期限切れ、ブラウザログイン状態消失。

**对策**：RexCLIはブラウザプロフィール保存をサポート、定期的なログイン状態チェック。

### 落とし穴3：重複コンテンツ判定

同じ文案を複数のアカウントに投稿すると、重複コンテンツと認定される。

**对策**：文案バリアント生成、同じ意味を三つの言い方でowered轮流使用。
