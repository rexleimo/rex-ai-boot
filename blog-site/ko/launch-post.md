# 드디어 'AI 코딩 도구 중 뭐가 더 좋은지' 논쟁에 종지부를 찍는 도구를 만들었다

솔직히? 과거에는 세 AI 코딩 도구에 완전히 휘둘리고 있었다.

Claude Code는 코딩은 잘 하지만 브라우저 자동화 설정이 귀찮다.
Codex 자동화는 만족스러운데 복잡한 코드 리팩토링은 부족하다.
Gemini는 연구에는 좋지만 실제로 사용하면 삽질한다.

"하나의 도구로 모든 걸 할 수 없나?" — 계속 생각했다.

그래서 직접 만들었다.

---

## 01. 어떤 문제에 부딪혔었나?

### 시나리오 1: 브라우저 자동화

소홍서에 자동 게시하고 싶었다.

Claude Code를 사용? 좋아, 먼저 MCP를 설정해. 3시간 후, 겨우 작동한다. 다음 날 아침에 일어났더니 — 계정이 플래그되었다.

Codex를 사용? 꺼내서 바로, 훌륭하다. 그런데中途、Claude에게 코드를 최적화해달라 했는데 — 도구를 전환한 순간 — **모든 컨텍스트 사라짐**, 처음부터 다시.

### 시나리오 2: 긴 작업中断

코드 리팩토링 작업을 실행 중, 2000줄 이상. 회의가 있어, 돌아와서 계속하려고 했는데 —

Claude: 대충 어디 있었는지 기억하지만 세부사항은 잊었다.
Codex: 대충 어디 있었는지 기억하지만 리팩토링 방향이 빗나갔다.
Gemini: 나는 누구, 여긴 어디?

**이것이 매일의 삶이었다: 도구를 전환하거나 모두를再設定하거나.**

---

## 02. RexCLI란 무엇인가?

**RexCLI = Claude Code + Codex + Gemini가協力動作, 서로 싸우지 않는다.**

### 핵심 기능

| 기능 | 의미 |
|-----------|---------------|
| 통일 브라우저 자동화 | 어떤 CLI든 같은 `browser_*` 명령어 |
| 크로스 CLI 컨텍스트 기억 | 도구 전환해도 진행 상황을 잃지 않음 |
| Privacy Guard | 설정 파일을 자동 삭제, API 키 유출 방지 |

### 동작 원리

```
あなたがcodex/claude/geminiと入力
       ↓
RexCLI가 자동 가로챈다
       ↓
판단: 래핑할까直通할까?
       ↓
래핑: ContextDB + Browser MCP를 연결
통과: 네이티브 도구에 전달
```

**습관을 바꿀 필요 없다.** 이전과 동일한 명령어를 사용하면 된다.

## 03. 실제 결과

### 결과 1: 브라우저 자동화

```bash
# 이전: Codex 사용
codex

# 现在: Codex 사용 중이지만 초월적 능력 보유
codex
```

차이는 통일된 `browser_*` 도구가 생긴 것이다:
- `browser_navigate` — 페이지 열기
- `browser_click` — 요소 클릭
- `browser_snapshot` — 페이지 콘텐츠 가져오기
- `browser_screenshot` — 스크린샷 찍기

**어떤 CLI로 전환하든 이 명령어들은 작동한다.**

### 결과 2: 브레이크포인트에서 재개

작업中途에 도구를 전환하고 싶으면?

```bash
# Codex로 절반 실행
codex

# Claude로 전환하여 계속, 컨텍스트 자동 동기화
claude
```

**복사 붙여넣기 불필요, 작업 재설명 불필요.**

## 05. 왜 오픈소스인가?

무슨 생각인지 안다: 이렇게 도구들이 많은데 왜 또 만드는가?

**도구의 분절화로 인한 고통을 저만큼 이해하는 사람은 없기 때문이다.**

매일 세 가지 도구를 전환하며 — 컨텍스트를 잃고, 설정을 중복하고, 키 유출 위험에 시달리며… 这个问题困扰我很久了.

참는 것보다 해결책을 만들었다.

**RexCLI는저의개인프로젝트로，매일사용하는 도구다。**

**RexCLI는 Claude Code나 Codex를 교체하려는 것이 아니다.**

같은 고통을 느낀다면해보아라. 도움이 됐다면, 좋아요와 공유 — 더 많은 사람이 혜택을받을 수 있도록.
