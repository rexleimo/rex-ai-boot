# Superpowers

Superpowersは再利用可能な自動化スキル。Claude Code、Codex、Gemini CLI、OpenCodeにフックして、反復タスクを自動処理する。

## Superpowersとは？

同じコマンドやプロンプトを繰り返す代わりに、スキルを起動すれば：
- AIを実証済みのフローに導く
- ベストプラクティスを自動強制
- 完了前に結果を検証

## あるスキル

### brainstorming

創造的な作業を始める前に、意図を明確にする。

- プロジェクトコンテキストを探索
- 明確な質問を一 つずつ
- アプローチ案とトレードオフを提案
- コード前にデザイン提示と承認

**使うとき**：新機能開発、ページデザイン、機能追加。

### writing-plans

要件を実行可能な計画に変換。

- 要件を分析
- 順序付きステップに分解
- 依存関係を特定
- 詳細な計画ドキュメント出力

**使うとき**：仕様や複数ステップタスクがあり、ロードマップが必要なとき。

### verification-before-completion

検証なしで完了と言わない。

- 検証コマンドを実行
- 出力が期待通りか確認
- 具体的な証拠を持って初めて成功

**使うとき**：機能完了、バグ修正、PR作成前。

### systematic-debugging

推測ではなく、証拠でバグを直す。

- 症状とエラーメッセージを収集
- 仮説を形成
- 体系的にテスト
- 修正が有効か検証

**使うとき**：テスト失敗、クラッシュ、予期しない動作に出くわした時。

### dispatching-parallel-agents

複数の独立タスクを並行実行。

- 独立したワークフローを見極め
- 並列agentsを起動
- 結果を統合
- 失敗を適切に処理

**使うとき**：2つ以上の共有状態を持たないタスクを同時実行したい時。

### security-scan

自動化前に設定のセキュリティ問題をチェック。

- skills、hooks、MCP設定をスキャン
- 露出したシークレットを特定
- 修正を提案

**使うとき**：自動化有効化や設定変更時。

## 使い方

1. Superpowersが必要なら、自然に言う
2. AIがスキルを起動してナビゲート
3. 結果はプロジェクトメモリに保存

## 例

```
brainstormingでこの機能怎么做/
writing-plansでこの要件を步骤に分ける
verification-before-completionで完成前に検証
```
